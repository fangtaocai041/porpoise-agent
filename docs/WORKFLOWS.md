# 🔄 工作流设计

## 1. 典型工作流

### 1.1 文献综述工作流

```
用户: "综述近五年长江江豚被动声学监测研究进展"
  │
  ▼
[Phase 0: 问题理解]
  → 关键词路由: "综述" + "声学监测" → Phase 1 文献调研
  │
  ▼
[Phase 1: 文献调研]
  1. 问题拆解 (3个子问题)
  2. 多源检索 (arXiv, PubMed, Scholar)
  3. 相关性筛选 (高/中/低)
  4. 深度阅读 (高相关文献)
  5. 交叉引用追踪
  6. 生成综述草稿
  │
  ▼
[Phase 5: 报告生成]
  → 结构化 Markdown 综述
  → 引用列表 (APA 格式)
```

### 1.2 声学数据分析工作流

```
用户: "分析2024年3月安庆江段PAM数据"
  │
  ▼
[Phase 0: 问题理解]
  → 关键词路由: "PAM数据" → Phase 2 数据分析
  │
  ▼
[Phase 2: 数据分析]
  1. 加载数据 (SoundTrap .wav, 576 kHz)
  2. 预处理 (带通滤波 100-180 kHz)
  3. Click 检测 (SPL阈值 -134 dB)
  4. Click Train 提取 (ICI模式匹配)
  5. Buzz 检测 (ICI < 10 ms)
  6. 特征提取 (18+ 特征)
  7. 分类 (Regular/Buzz/Noise/Vessel)
  8. 行为推断 (觅食活动指数)
  9. 昼夜节律分析
  │
  ▼
[Phase 5: 报告生成]
  → 监测报告 (含图表)
```

### 1.3 野外调查规划工作流

```
用户: "规划2025年春季鄱阳湖江豚调查"
  │
  ▼
[Phase 0: 问题理解]
  → 关键词路由: "调查" → Phase 3 野外调查
  │
  ▼
[Phase 3: 野外调查]
  1. 历史数据分析 (已有观测/声学数据)
  2. 样线设计 (Distance Sampling原理)
  3. 设备部署建议 (SoundTrap/A-tag/C-POD)
  4. 调查日程安排
  5. 安全注意事项
  6. 数据管理计划
  │
  ▼
  ⚠️ 人工审批关卡
  │
  ▼
[Phase 5: 报告生成]
  → 调查方案书
```

### 1.4 保护评估工作流

```
用户: "评估安庆江段江豚保护状况"
  │
  ▼
[Phase 1: 文献调研] → 背景资料
  │
  ▼
[Phase 2: 数据分析] → PAM数据 + 目视数据
  │
  ▼
[Phase 4: 保护评估]
  1. IUCN标准评估
  2. 威胁因子排序
  3. 栖息地质量评估
  4. 保护行动优先级
  5. 管理建议
  │
  ▼
  ⚠️ 人工审批关卡 (管理建议)
  │
  ▼
[Phase 5: 报告生成]
  → 保护评估报告
  → 管理建议书
```

## 2. 工作流状态机

```
                    ┌──────────┐
                    │  START   │
                    └────┬─────┘
                         │
                    ┌────▼─────┐
                    │ Phase 0  │ 问题理解与路由
                    └────┬─────┘
                         │
          ┌──────────────┼──────────────┬──────────────┐
          ▼              ▼              ▼              ▼
    ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐
    │ Phase 1  │  │ Phase 2  │  │ Phase 3  │  │ Phase 4  │
    │ 文献调研  │  │ 数据分析  │  │ 野外调查  │  │ 保护评估  │
    └────┬─────┘  └────┬─────┘  └────┬─────┘  └────┬─────┘
         │              │              │              │
         └──────────────┼──────────────┴──────────────┘
                        │
                   ┌────▼─────┐
                   │ Phase 5  │ 成果产出
                   └────┬─────┘
                        │
                   ┌────▼─────┐
                   │   END    │
                   └──────────┘
```

## 3. 人工审批节点

| 节点 | 触发条件 | 审批内容 |
|------|----------|----------|
| 野外方案审批 | Phase 3 完成 | 调查路线、设备部署、安全措施 |
| 保护建议审批 | Phase 4 完成 | 管理建议的可行性和政策对齐 |
| 数据删除 | 用户请求删除数据 | 二次确认，防止误删 |
| 外部API写入 | 任何写入操作 | 确认发送内容 |
